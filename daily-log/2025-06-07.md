# Day1 -v4: 状態・履歴確認の習慣化

## 学習内容

### 1. ブランチの確認と削除
- **コマンド**: `git branch -a`
  - ローカルとリモートの全ブランチを一覧表示
  - 現在のブランチは `*` で示される

- **不要なブランチの削除**
  - ローカルブランチ: `git branch -d <ブランチ名>`
  - リモートブランチ: `git push origin --delete <ブランチ名>`
  - 複数ブランチの一括削除: `git branch | grep "day" | xargs git branch -d`

### 2. `git fetch` と `git pull` の違い
- **`git fetch`**
  - リモートの最新情報を取得するだけで、ローカルの状態は変更しない
  - リモートとローカルの差分を確認するのに役立つ

- **`git pull`**
  - リモートの最新情報を取得し、自動的にローカルのブランチにマージする
  - リモートの最新情報をすぐにローカルに反映したい場合に使用

### 3. `git log --oneline --graph --decorate --all` の読み方
- **出力の意味**
  - コミットハッシュ: 各行の先頭にある文字列（例: `9ca257d`）
  - ブランチ名・タグ名: `(HEAD -> main, origin/main, origin/HEAD)` など
  - コミットメッセージ: ハッシュの後に続く文字列
  - グラフィカルな表示: `*` はコミット、`|` や `\` はブランチの分岐やマージ

- **トラブルシューティング**
  - マージやリベースの失敗
  - ブランチの分岐やマージの混乱
  - コミットの履歴の混乱
  - リモートとローカルの差分
  - バグの原因特定

## 振り返り
- 状態・履歴確認の習慣化が重要
- トラブル発生時は、`git log` や `git status` で状況を把握する
- リモートとローカルの差分を常に確認する
